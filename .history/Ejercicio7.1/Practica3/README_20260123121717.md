EXPLICACIÓN DEL PROBLEMA 

Cada vez que se renderiza ToastContainer, se crean nuevas funciones inline para la prop onClose. Esto provoca que React considere que las props de todos los ToastItem han cambiado, forzando su re-renderizado incluso aunque el error no haya cambiado.


![alt text](image.png)


¿Qué cambios se han hecho y por qué?

Se ha utilizado useCallback para memorizar las funciones que se pasan como props y evitar que se creen nuevas referencias en cada render. Además, se ha aplicado React.memo a ToastContainer y ToastItem para evitar renderizados innecesarios. De este modo, solo se vuelve a renderizar el toast que realmente cambia o desaparece, mejorando significativamente el rendimiento.
