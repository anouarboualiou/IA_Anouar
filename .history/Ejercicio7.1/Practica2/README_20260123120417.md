¿Por qué va lenta?

    La aplicación es lenta porque en cada render se ejecuta un filtro sobre una lista de 10.000 usuarios. Esto provoca que UserList y todos los componentes UserCard se rendericen de nuevo en cada pulsación del teclado, aunque sus datos no hayan cambiado. El React Profiler muestra un número elevado de renders innecesarios y un tiempo de commit alto.

¿Ha mejorado? ¿Qué cambios y por qué?

    Tras la optimización, el rendimiento de la aplicación ha mejorado notablemente. Se ha utilizado useMemo para memorizar el resultado del filtrado de usuarios, evitando recalcular una operación costosa en cada render.
    Además, se ha aplicado React.memo a los componentes UserList, UserCard y SearchInput, ya que son componentes puros cuyos props no cambian innecesariamente.
    El React Profiler muestra una reducción significativa del número de renders y del tiempo de commit, haciendo la aplicación mucho más fluida.
